# Slicer Configuration Keys

This document lists common configuration keys that can be used effectively with `slicer_load_preset_with_overrides` (or `custom_config_json` in Rust). These keys correspond to settings in BambuStudio/Slic3r.

## Common Overrides

### Quality & Layers
- `layer_height`: (float) Layer height in mm (e.g., 0.2).
- `first_layer_height`: (float) First layer height in mm (e.g., 0.2).
- `initial_layer_line_width_factor`: (float) Factor for initial layer line width (e.g., 105%).

### Infill
- `sparse_infill_density`: (string) Infill density percentage (e.g., "15%").
- `sparse_infill_pattern`: (string) Infill pattern (e.g., "grid", "gyroid", "adaptivecubic").
- `infill_anchor`: (string) Infill anchor (e.g., "600%").
- `infill_anchor_max`: (string) Infill anchor max (e.g., "50").

### Speed
- `outer_wall_speed`: (float) Speed for outer walls in mm/s.
- `inner_wall_speed`: (float) Speed for inner walls in mm/s.
- `sparse_infill_speed`: (float) Speed for sparse infill in mm/s.
- `internal_solid_infill_speed`: (float) Speed for internal solid infill in mm/s.
- `top_surface_speed`: (float) Speed for top surface in mm/s.
- `gap_infill_speed`: (float) Speed for gap infill in mm/s.
- `initial_layer_speed`: (float) Speed for initial layer in mm/s.

### Support
- `enable_support`: (bool) Enable/disable support generation.
- `support_type`: (string) Type of support (`normal(auto)`, `tree(auto)`, `normal(manual)`, `tree(manual)`).
- `support_style`: (string) Style of support (`default`, `grid`, `snug`, `tree_slim`, `tree_strong`, `tree_hybrid`, `tree_organic`).
- `support_threshold_angle`: (int) Overhang threshold angle (e.g., 30).
- `support_on_buildplate_only`: (bool) Generate support only on build plate.

### Perimeter / Walls
- `wall_loops`: (int) Number of wall loops.
- `detect_thin_walls`: (bool) Detect thin walls.
- `seam_position`: (string) Seam position (`scrambled`, `nearest`, `aligned`, `rear`).

### Other
- `skirt_loops`: (int) Number of skirt loops.
- `brim_type`: (string) Brim type (`no_brim`, `outer_only`, `inner_only`, `outer_and_inner`, `auto_brim`, `brim_ears`).
- `brim_width`: (float) Brim width in mm.

> [!TIP]
> **Note on Values:** Most values can be passed as strings, even if the internal type is float or int, as the deserializer handles conversion. However, sticking to the expected type in JSON is recommended.
